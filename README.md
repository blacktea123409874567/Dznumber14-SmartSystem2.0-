При обычном множественном наследовании класс может получить от «родителей» два разных варианта одного и того же метода, что приводит к ошибкам.
Интерфейсы не дают готовый код, а только требуют его написать. В итоге класс сам определяет логику, и никакой путаницы между родительскими методами не возникает.
